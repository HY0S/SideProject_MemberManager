---

## 1. 프로그램 개요

| 항목 | 내용 |
|------|------|
| **프로그램명** | 홀덤펍 유저 관리 프로그램 |
| **기획 의도** | 홀덤펍 현장에서 고객(유저)들의 점수와 빙고 이벤트 현황을 효율적으로 관리 |
| **개발 목적** | 수기/엑셀 기반 관리의 불편함 해소, 관리자·알바생 모두가 쉽게 사용할 수 있는 전용 관리 프로그램 제공 |

| **작성자 / 작성일** | _(미기입)_ |

## 2. 기획 배경 및 필요성

- 홀덤펍 현장에서 고객(유저)의 게임 참여도·이벤트 진행 상황을 체계적으로 관리할 필요성
- 현재 고객 점수·빙고 이벤트를 수기 또는 비정형 방식으로 관리하여 현황 파악이 어려움
- 빙고 이벤트 특성상 참여 고객이 많아질수록 관리 부담과 오류 가능성 증가
- 고객 점수·빙고 현황을 직관적으로 관리할 수 있는 전용 프로그램 필요

---

## 3. 목표 및 기대 효과

### 3.1 목표

- 직원(회원)별 점수 관리 기능 제공
- 직원별 빙고 판 및 진행 현황 저장
- 관리자 중심의 간단한 조작 UI 제공

### 3.2 기대 효과

- 점수/빙고 관리 시간 단축
- 데이터 누락 및 오류 최소화
- 이벤트 참여율 및 직원 동기 부여 향상

---

## 4. 대상 사용자 분석

| 구분 | 설명 |
|------|------|
| **관리 대상(회원)** | 홀덤펍 고객 |
| **주 사용자** | 관리자(점장, 매니저) |
| **부 사용자** | 알바생(조회 및 보조 관리) |

### 사용 시나리오

1. 프로그램 실행 후 관리자 비밀번호 입력
2. 고객(닉네임) 등록
3. 게임/이벤트 참여 시 점수 버튼으로 점수 추가
4. 빙고 이벤트 진행 시 고객을 빙고판 칸에 배치
5. 빙고 줄 수 및 올빙고 여부 자동 확인
6. 모든 데이터는 파일로 자동 저장

---

## 5. 프로그램 주요 기능

| 번호 | 기능명 | 기능 설명 | 우선순위 |
|------|--------|------------|----------|
| 1 | 고객 관리 | 고객 닉네임 등록 및 삭제 | 높음 |
| 2 | 점수 관리 | 특정 행동 시 버튼 클릭으로 점수 추가 | 높음 |
| 3 | 빙고 관리 | 고객을 빙고 칸에 배치하여 빙고 현황 계산 | 높음 |
| 4 | 빙고 판 설정 | 3×3 기본, 2×2 ~ 6×6까지 가변 설정 | 중간 |
| 5 | 현황 조회 | 고객별 점수, 빙고 줄 수 및 올빙고 여부 확인 | 중간 |
| 6 | 데이터 저장 | 파일 기반 자동 저장 및 불러오기 | 높음 |

## 6. 화면 구성 및 동작 흐름

### 6.1 전체 흐름

**프로그램 실행 → 메인 화면 표시 → 기능 선택 → 데이터 입력/조회 → 자동 저장**

### 6.2 화면 구성

| 화면명 | 설명 |
|--------|------|
| 메인 화면 | 회원 목록 및 요약 정보 표시 |
| 회원 관리 화면 | 회원 추가/수정/삭제 |
| 점수 관리 화면 | 점수 입력 및 수정 |
| 빙고 화면 | 빙고판 표시 및 체크 |

## 7. 시스템 구조

### UI 레이어

- Java 데스크톱 UI (Swing 또는 JavaFX)

### 도메인 로직 레이어

- 고객 관리 로직
- 행동 기반 점수 누적 로직
- 가변 크기 빙고 판 관리 로직
- 가로/세로/대각선 빙고 판정 로직
- 올빙고(모든 칸 입력) 판정 로직

### 데이터 레이어

- JSON 파일 기반 저장
- 저장 시 백업 파일 자동 생성
- 점수 데이터와 빙고 데이터 분리 저장을 통한 무결성 확보

---

## 8. 데이터 설계

### 고객 데이터

- 닉네임
- 누적 점수

### 빙고 데이터

- 빙고 판 크기(행×열)
- 각 칸에 배치된 고객 닉네임
- 고객별 완성된 빙고 줄 수

### 저장 방식

- JSON 파일을 이용한 로컬 저장
- 프로그램 재실행 시 이전 상태 복원 가능

## 13. 결론

- 본 프로그램은 홀덤펍 현장에서 고객 점수 및 빙고 이벤트를 안정적으로 관리하는 것을 목표로 함
- JSON + 날짜별 백업 파일 전략을 통해 **데이터 무결성**을 최우선으로 보장함
- 변경 시 즉시 저장과 오류 대응 설계를 통해 실제 매장 환경에서도 안정적으로 사용 가능함
- **1차 완성 기준**: 실제 매장에서 하루 이상 사용 가능 여부로 정의함
